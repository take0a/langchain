# はじめに

**LangChain** は、大規模言語モデル (LLM) を活用したアプリケーション開発フレームワークです。

LangChain は、LLM アプリケーションライフサイクルのあらゆる段階を簡素化します。
- **開発**: LangChain のオープンソース [コンポーネント](/docs/concepts) と [サードパーティ統合](/docs/integrations/providers/) を使用してアプリケーションを構築します。
- [LangGraph](/docs/concepts/architecture/#langgraph) を使用して、最高クラスのストリーミングと人間参加型サポートを備えたステートフルエージェントを構築します。
- **本番環境化**: [LangSmith](https://docs.smith.langchain.com/) を使用してアプリケーションを検査、監視、評価することで、自信を持って継続的に最適化とデプロイを行うことができます。
- **デプロイメント**: [LangGraph プラットフォーム](https://langchain-ai.github.io/langgraph/cloud/) を使用して、LangGraph アプリケーションを本番環境対応の API およびアシスタントに変換します。

## アーキテクチャ

LangChainフレームワークは、複数のオープンソースライブラリで構成されています。詳しくは[アーキテクチャ](/docs/concepts/architecture/)ページをご覧ください。

- **`langchain-core`**: チャットモデルやその他のコンポーネントのための基本抽象化。
- **統合パッケージ** (例: `langchain-openai`、`langchain-anthropic` など): 重要な統合機能は、LangChainチームと統合開発者が共同でメンテナンスする軽量パッケージに分割されています。
- **`langchain`**: アプリケーションのコグニティブアーキテクチャを構成するチェーン、エージェント、および検索戦略。
- **`langchain-community`**: コミュニティによってメンテナンスされているサードパーティ統合。
- **`langgraph`**: LangChainコンポーネントを、永続性、ストリーミング、その他の主要機能を備えた本番環境対応アプリケーションに統合するためのオーケストレーションフレームワーク。 [LangGraph ドキュメント](https://langchain-ai.github.io/langgraph/)を参照してください。

## Ecosystem

### [🦜🛠️ LangSmith](https://docs.smith.langchain.com)
言語モデル アプリケーションとインテリジェント エージェントをトレースおよび評価して、プロトタイプから本番環境への移行を支援します。

### [🦜🕸️ LangGraph](https://langchain-ai.github.io/langgraph)
LLMを用いて、ステートフルなマルチアクターアプリケーションを構築できます。LangChainとのスムーズな統合はもちろん、LangChainなしでも利用可能です。LangGraphは、Linkedin、Uber、Klarna、GitLabなど、多くの企業で信頼されているプロダクショングレードのエージェントを支えています。
