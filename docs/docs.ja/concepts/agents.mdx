# エージェント

言語モデル自体はアクションを実行できず、テキストを出力するだけです。エージェントは、高レベルのタスクを受け取り、LLMを推論エンジンとして使用して、実行するアクションを決定し、実行するシステムです。

[LangGraph](/docs/concepts/architecture#langgraph) は、高度な制御性とカスタマイズ性を備えたエージェントの作成に特化したLangChainの拡張機能です。エージェントの構築にはLangGraphの使用をお勧めします。

詳細については、以下のリソースをご覧ください。

* [一般的なエージェントアーキテクチャ](https://langchain-ai.github.io/langgraph/concepts/agentic_concepts/) に関するLangGraphのドキュメント
* [LangGraphのプレビルドエージェント](https://langchain-ai.github.io/langgraph/reference/prebuilt/#langgraph.prebuilt.chat_agent_executor.create_react_agent)

## レガシーエージェントのコンセプト：AgentExecutor

LangChainは以前、エージェントのランタイムとして「AgentExecutor」を導入していました。
これは優れた出発点となりましたが、より高度でカスタマイズされたエージェントを扱う際には限界が明らかになりました。
そのため、LangGraphでは「AgentExecutor」を段階的に廃止し、より柔軟なソリューションを導入していく予定です。

### AgentExecutor から LangGraph への移行

現在 `AgentExecutor` をご利用の方もご安心ください。移行に役立つリソースをご用意しています。

1. 引き続き `AgentExecutor` をご利用いただく必要がある方のために、[AgentExecutor の使い方](/docs/how_to/agent_executor) に関する包括的なガイドをご用意しています。

2. ただし、柔軟性と制御性を向上させるために、LangGraph への移行を強くお勧めします。この移行を容易にするために、詳細な [移行ガイド](/docs/how_to/migrate_agent) を作成しました。このガイドは、`AgentExecutor` から LangGraph へのシームレスな移行を支援します。
