LangChain は、複数のパッケージで構成されるフレームワークです。

## langchain-core

このパッケージには、さまざまなコンポーネントの基本抽象化と、それらを組み合わせる方法が含まれています。
チャットモデル、ベクターストア、ツールなどのコアコンポーネントのインターフェースはここで定義されています。
サードパーティとの統合はここで定義されていません。
依存関係は非常に軽量です。

## langchain

メインの`langchain`パッケージには、アプリケーションの認知アーキテクチャを構成するチェーンと検索戦略が含まれています。
これらはサードパーティ統合ではありません。
ここで説明するすべてのチェーン、エージェント、および検索戦略は、特定の統合に固有のものではなく、すべての統合に共通です。

## Integration packages

一般的な統合には独自のパッケージ（例：`langchain-openai`、`langchain-anthropic` など）が用意されており、適切なバージョン管理と軽量化が図られています。

詳細については、以下をご覧ください。

* [統合パッケージ](/docs/integrations/providers/)の一覧
* 各統合パッケージの詳細情報が記載されている [API リファレンス](https://python.langchain.com/api_reference/)

## langchain-community

このパッケージには、LangChainコミュニティによってメンテナンスされているサードパーティ統合が含まれています。
主要な統合パッケージは分離されています（上記参照）。
このパッケージには、チャットモデル、ベクターストア、ツールなど、様々なコンポーネントの統合が含まれています。
このパッケージ内のすべての依存関係は、パッケージを可能な限り軽量に保つためにオプションです。

## langgraph

`langgraph` は `langchain` の拡張版であり、ステップをグラフ内のエッジとノードとしてモデル化することで、LLM を用いた堅牢でステートフルなマルチアクターアプリケーションの構築を目指しています。

LangGraph は、一般的なエージェントを作成するための高レベルインターフェースと、カスタムフローを作成するための低レベル API を公開しています。

:::info[参考資料]

* LangGraphの概要については[こちら](https://langchain-ai.github.io/langgraph/concepts/high_level/#core-principles)をご覧ください。
* LangGraphアカデミーコースについては[こちら](https://academy.langchain.com/courses/intro-to-langgraph)をご覧ください。
:::

## langserve

LangChainチェーンをREST APIとしてデプロイするためのパッケージ。本番環境に対応したAPIを簡単に立ち上げ、実行できます。

:::important
LangServeは、主にシンプルなRunnableをデプロイし、langchain-coreのよく知られたプリミティブと連携するように設計されています。

LangGraphのデプロイオプションが必要な場合は、LangGraphアプリケーションのデプロイに適したLangGraph Platform（ベータ版）をご検討ください。
:::

詳細については、[LangServe のドキュメント](/docs/langserve)を参照してください。


## LangSmith

LLM アプリケーションのデバッグ、テスト、評価、監視を可能にする開発プラットフォームです。

詳細については、[LangSmith ドキュメント](https://docs.smith.langchain.com) をご覧ください。
