# チュートリアル

LangChain や LLM アプリ開発全般を初めてご利用ですか？この資料を読んで、すぐに最初のアプリケーションの構築を開始しましょう。

## 始めましょう

シンプルなアプリケーションを構築することで、LangChainのオープンソースコンポーネントに慣れましょう。

[チャットモデル](/docs/integrations/chat/)、[ベクターストア](/docs/integrations/vectorstores/)、または特定のプロバイダーのその他のLangChainコンポーネントの使用を開始する場合は、サポートされている[統合](/docs/integrations/providers/)をご確認ください。

- [チャットモデルとプロンプト](/docs/tutorials/llm_chain): [プロンプトテンプレート](/docs/concepts/prompt_templates)と[チャットモデル](/docs/concepts/chat_models)を使用して、シンプルなLLMアプリケーションを構築します。
- [セマンティック検索](/docs/tutorials/retrievers): [ドキュメントローダー](/docs/concepts/document_loaders)、[埋め込みモデル](/docs/concepts/embedding_models/)、[ベクターストア](/docs/concepts/vectorstores/)を使用して、PDFを対象としたセマンティック検索エンジンを構築します。
- [分類](/docs/tutorials/classification): [チャットモデル](/docs/concepts/chat_models)と[構造化出力](/docs/concepts/structured_outputs/)を使用して、テキストをカテゴリまたはラベルに分類します。
- [抽出](/docs/tutorials/extraction): [チャットモデル](/docs/concepts/chat_models)と[few-shot examples](/docs/concepts/few_shot_prompting/)を使用して、テキストやその他の非構造化メディアから構造化データを抽出します。

すべての LangChain コンポーネントの使用に関する詳細については、[ハウツー ガイド](/docs/how_to) を参照してください。

## オーケストレーション

[LangGraph](https://langchain-ai.github.io/langgraph/) を使って、LangChain コンポーネントをフル機能のアプリケーションに組み立ててみましょう。

- [チャットボット](/docs/tutorials/chatbot): メモリを組み込んだチャットボットを構築します。
- [エージェント](/docs/tutorials/agents): 外部ツールと連携するエージェントを構築します。
- [検索拡張生成 (RAG) パート 1](/docs/tutorials/rag): 独自のドキュメントに基づいて応答を生成するアプリケーションを構築します。
- [検索拡張生成 (RAG) パート 2](/docs/tutorials/qa_chat_history): ユーザーインタラクションのメモリと複数ステップの検索を組み込んだ RAG アプリケーションを構築します。
- [SQL による質問応答](/docs/tutorials/sql_qa): SQL クエリを実行して応答を生成する質問応答システムを構築します。
- [要約](/docs/tutorials/summarization): （長くなる可能性のある）テキストの要約を生成します。
- [グラフデータベースによる質問応答](/docs/tutorials/graph): グラフデータベースにクエリを実行して応答を生成する質問応答システムを構築します。

## LangSmith

LangSmith を使用すると、LLM アプリケーションを詳細にトレース、監視、評価できます。
LangSmith は LangChain とシームレスに統合されており、ビルド時にチェーンの各ステップを検査およびデバッグできます。

LangSmith のドキュメントは別のサイトでホストされています。
[LangSmith のチュートリアルはこちら](https://docs.smith.langchain.com/) でご覧いただけます。

### 評価

LangSmith は、LLM アプリケーションのパフォーマンス評価に役立ちます。以下のチュートリアルは、評価を始めるのに最適な方法です。
